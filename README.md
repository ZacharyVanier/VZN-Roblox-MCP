# VZN Roblox MCP

**Multi-Studio Roblox MCP Server** by Zachary Vanier (VZNZach)

A Model Context Protocol (MCP) server that connects to **multiple Roblox Studio instances** simultaneously via a single HTTP bridge. Auto-detects all connected Studios and lets you choose which one to control.

## Features

- Single HTTP server with plugin polling architecture
- Auto-discovers and registers multiple Studio instances
- Auto-selects if only one Studio is connected
- Prompts for selection when multiple Studios are detected
- Execute Luau code with output capture (prints show in Studio Output)
- Read/write script source code via ScriptEditorService
- Create, delete, and modify instances
- Search for instances by name or class
- Get instance properties and children
- Exponential backoff and auto-reconnect on connection loss
- Enable/Disable toggle in the Studio plugin widget

## Architecture

```
Claude Code <--stdio--> MCP Server <--HTTP--> Studio Plugin(s)
```

1. The Node.js MCP server starts an HTTP server on localhost (port 3002-3020)
2. Each Studio plugin discovers the server, registers with a unique GUID
3. Plugins poll for commands every 0.5s
4. Claude sends commands via MCP tools, queued for the target Studio, plugin picks them up, executes, and returns results

## Setup

### 1. Clone and Build the Server

```bash
git clone https://github.com/ZacharyVanier/VZN-Roblox-MCP.git
cd VZN-Roblox-MCP
npm install
npm run build
```

### 2. Install the Studio Plugin

A pre-built plugin file is included at `plugin/McpPluginVZN.rbxmx`. Just copy it to your Roblox plugins folder:

| OS      | Plugins Folder                          |
|---------|-----------------------------------------|
| Windows | `%LOCALAPPDATA%\Roblox\Plugins\`        |
| macOS   | `~/Documents/Roblox/Plugins/`           |

**Windows:**
```bash
copy plugin\McpPluginVZN.rbxmx "%LOCALAPPDATA%\Roblox\Plugins\"
```

**macOS:**
```bash
cp plugin/McpPluginVZN.rbxmx ~/Documents/Roblox/Plugins/
```

You can also find the plugins folder from Studio: **Plugins tab -> Plugins Folder**, then drop the file there.

> **Rebuilding the plugin (optional):** If you modify the plugin source, rebuild it with [Rojo](https://rojo.space):
> ```bash
> cd plugin
> rojo build -o McpPluginVZN.rbxmx
> ```

### 3. Add to Claude

#### Claude Code (CLI)

```bash
claude mcp add --transport stdio vzn-roblox-mcp -- node /path/to/VZN-Roblox-MCP/dist/index.js
```

Or manually add to `~/.claude.json`:
```json
{
  "mcpServers": {
    "vzn-roblox-mcp": {
      "type": "stdio",
      "command": "node",
      "args": ["/full/path/to/VZN-Roblox-MCP/dist/index.js"]
    }
  }
}
```

#### Claude Desktop (App)

Edit the Claude Desktop config file:

| OS      | Config File                                                    |
|---------|----------------------------------------------------------------|
| Windows | `%APPDATA%\Claude\claude_desktop_config.json`                  |
| macOS   | `~/Library/Application Support/Claude/claude_desktop_config.json` |

Add this to the file (replace the path with where you cloned the repo):

```json
{
  "mcpServers": {
    "vzn-roblox-mcp": {
      "command": "node",
      "args": ["/full/path/to/VZN-Roblox-MCP/dist/index.js"]
    }
  }
}
```

If the file already has other MCP servers, just add `"vzn-roblox-mcp": { ... }` inside the existing `mcpServers` object. Then restart Claude Desktop.

### 4. Enable HTTP Requests in Studio

The plugin needs HTTP access to communicate with the MCP server on localhost.

In Roblox Studio: **Game Settings -> Security -> Allow HTTP Requests -> ON**

### 5. Verify

1. Open Roblox Studio with a place
2. Start Claude Code -- the MCP server starts automatically
3. You should see the VZN MCP widget in Studio showing **"Status: Connected"** in green
4. Try running: `list_studios` from Claude to see the connected Studio

## Plugin Widget

The plugin adds a small dock widget in Studio showing:
- **Status** -- Connected / Discovering / Disconnected / Disabled
- **Port** -- Which localhost port the server is on
- **Studio ID** -- Unique identifier for this Studio instance
- **Place name** -- The published name of the place
- **Version** -- Plugin version
- **Enable/Disable button** -- Toggle the MCP connection on or off without uninstalling

## Tools

| Tool | Description |
|------|-------------|
| `list_studios` | List all connected Studio instances |
| `select_studio` | Pick which Studio to send commands to |
| `execute` | Run Luau code in the selected Studio |
| `get_children` | Get children of an instance |
| `get_properties` | Get properties of an instance |
| `get_script_source` | Get script source code |
| `set_script_source` | Update script source code |
| `create_instance` | Create a new instance |
| `delete_instance` | Delete an instance |
| `set_property` | Set a property on an instance |
| `search` | Search for instances by name/class |

## Requirements

- Node.js 18+
- Roblox Studio with HTTP Requests enabled
